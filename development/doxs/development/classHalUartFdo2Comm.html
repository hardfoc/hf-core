Detailed Description

CRTP adapter that turns a BaseUart& into a transport the templated fdo2::Driver<UartT> can drive.

Notes
  • BaseUart::Read returns a status code only — not the actual byte count — so this adapter polls BytesAvailable() first (with the caller's timeout as the deadline for the first byte) then drains whatever is currently in the FIFO plus a small grace window for any in-flight bytes. PSUP lines are short (~80 chars; ~42 ms at 19200 baud) so the drain window is bounded.
  • All timing (delays, deadline tracking) goes through the hf-utils-rtos-wrap os_* primitives — the handler has zero direct ESP-IDF / FreeRTOS dependencies, so the same code path works against any RTOS the wrapper supports.
