Handler Pattern

Each handler follows a consistent pattern:

  1. Construction â€” Accepts base interface references (not owned)
  2. Lazy initialization â€” Initialize() or EnsureInitialized() creates the CRTP driver
  3. Thread safety â€” RtosMutex protects all public methods
  4. Error propagation â€” Maps driver errors to interface error codes
  5. Zero overhead â€” CRTP dispatch means no virtual function overhead in the driver layer

RTOS Integration

  • RtosMutex â€” Wrapper around xSemaphoreCreateMutex
  • MutexLockGuard â€” RAII lock guard
  • PeriodicTimer â€” Wrapper around FreeRTOS software timers
  • BaseThread â€” Abstract base for FreeRTOS tasks
  • OsQueue, OsEventFlags, OsSemaphore â€” Standard RTOS primitives

All handlers use RtosMutex internally. The MutexLockGuard RAII pattern ensures deadlock-free operation even when exceptions or early returns occur.
