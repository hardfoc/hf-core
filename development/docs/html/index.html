hf-core is the shared platform layer for all HardFOC boards. It provides the handler layer, RTOS utilities, and driver integration that form the backbone of the HardFOC motor control system.

hf-core is the bridge layer between hardware-agnostic base interfaces (BaseI2c, BaseSpi, BaseGpio, BaseAdc, BaseTemperature) and CRTP-templated device drivers (hf-tmc9660-driver, hf-bno08x-driver, hf-as5047u-driver, etc.). No board-specific configuration lives here â€” board HALs (e.g. hf-hal-vortex-v1) depend on this repo and add managers, API, and pin mapping.

Clone

git clone --recurse-submodules https://github.com/hardfoc/hf-core.git

Or after a shallow clone:

git submodule update --init --recursive

Build and Flash a Test

cd examples/esp32/scripts
# Build a utility test (no hardware needed)
./build_app.sh --app general_utils_test --build-type debug
# Flash and monitor
cd .. && idf.py -p /dev/ttyUSB0 flash monitor

Build All CI-Enabled Tests

./build_app.sh --all --build-type debug
