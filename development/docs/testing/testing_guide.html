Testing Guide

Overview

hf-core uses an ESP32-based testing infrastructure that mirrors the pattern used by all external HardFOC drivers. Each test is a standalone ESP-IDF application selected at build time via the APP_TYPE mechanism.

Pin Configuration

Pin assignments are defined in esp32_test_config.hpp and can be overridden with compile-time -D flags. See the file for the complete pin map.

Test Framework

All tests use TestFramework.h ‚Äî a shared header providing:

  • RUN_TEST(name, fn) ‚Äî Run a test function, track pass/fail
  • RUN_TEST_IN_TASK(name, fn, stack, timeout) ‚Äî Run in a FreeRTOS task with timeout
  • RUN_TEST_SECTION_IF_ENABLED(flag, name, ...) ‚Äî Conditionally run test groups
  • flip_test_progress_indicator() ‚Äî Toggle GPIO14 for visual progress
  • print_test_summary(results, name, tag) ‚Äî Print final pass/fail report

#include "TestFramework.h"
static TestResults g_test_results;

static bool test_my_feature() noexcept {
    // ... test logic ...
    return success;
}

extern "C" void app_main(void) {
    RUN_TEST("my_feature", test_my_feature);
    print_test_summary(g_test_results, "MY TEST SUITE", "MyTag");
    while (true) { vTaskDelay(pdMS_TO_TICKS(10000)); }
}

Then add the app to app_config.yml and update main/CMakeLists.txt.
